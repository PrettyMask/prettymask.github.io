键盘弹起之后的页面错乱问题

Posted on 2019-08-19

背景

如果使用了 fixed 布局的话，键盘弹起，fixed 布局会被替换成 absolute 布局，布局被顶起，键盘收起的时候也不会复原，需要点一下其他区域才能复原，用户体验差。

代码解决

不谈如何绕过这个问题，直面问题。

思路：

  1. 失去聚焦的时候让布局恢复原位
  2. 有多个输入框的时候，来回聚焦/失焦会产生抖动，所以加个延时

以下是 react 的写法。

说下 window.scrollTo()

第一种

参数：
  • x-coord 是文档中的横轴坐标。
  • y-coord 是文档中的纵轴坐标。
    兼容性
    兼容性良好，基本所有浏览器支持

第二种

参数
  • options 是一个包含三个属性的对象:
    • top 等同于 y-coord
    • left 等同于 x-coord
    • behavior 类型String,表示滚动行为,支持参数 smooth(平滑滚动),instant(瞬间滚动),默认值auto,实测效果等同于instant
  • 示例
兼容性

兼容性较差，safari 不支持，edge 不支持，其他浏览器的新版本支持

  1. flex和inline-flex区别在于，inline-flex容器为inline特性，因此可以和图片文字一行显示；flex容器保持块状特性，宽度默认100%，不和内联元素一行显示

  2. 无论作用在flex容器上，还是作用在flex子项，都是控制的flex子项的呈现，只是前者控制的是整体，后者控制的是个体。
